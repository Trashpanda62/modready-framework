## 4. In-progress: "every mod on one page" rewrite

### Goal

Replace the fixed-slot pagination (`SlotCount = 20`, `Page X of Y` controls) in the Mod Configuration tab with an infinitely scrollable list. Every mod gets one page; user scrolls vertically through all its settings regardless of count.

### Slices

**4a ŌĆö DONE this session.** New file `src/ModReady.MCM/UI/PrefabExtensions/PresentationRowVM.cs`. Single-row ViewModel that holds EITHER a `SettingsPropertyVM` (real settings row) OR a header-text string (group divider). Mirrors the entire `Slot0_*` binding surface 1:1 (same names, types, behaviors): `IsVisible`, `IsHeader`, `IsProperty`, `GroupHeader`, `DisplayName`, `HintText`, `IsBool` / `IsInteger` / `IsFloating` / `IsNumeric` / `IsText` / `IsButton` / `IsDropdown`, `MinValue` / `MaxValue`, `BoolValue` + `BoolText`, `ButtonText`, `IntValue` / `FloatValue` / `ValueText` / `EditableValueText` (with parse + clamp setter), `TextValue`, `DropdownText`, `ExecuteToggleBool` / `ExecuteDropdownNext` / `ExecuteDropdownPrev` / `ExecuteAction`, `NotifyRefresh()` helper for post-preset-Apply re-paint. Uses `Bannerlord.UIExtenderEx.Attributes` for `[DataSourceMethod]` and `TaleWorlds.Library` for `[DataSourceProperty]`.

**4b ŌĆö DONE this session.** `OptionsVMMixin.cs`. Added private `MBBindingList<PresentationRowVM> _rowList`, exposed as `[DataSourceProperty] public MBBindingList<PresentationRowVM> RowList`. `SelectMod` clears `_rowList` and rebuilds it from `_presentation` (the existing per-mod flattened list, one row per group-header or property) in parallel with the legacy slot fan. The old `Slot0..Slot19` data-source properties + their fan are untouched and still drive the current XML ŌĆö `RowList` is purely additive so far.

**Slice 1 (diagnostic, separate from 4a-d) ŌĆö DONE.** `src/ModReady.MCM/Internal/PresentationRowSurvey.cs`. Walks `SettingsRegistry.All`, mirrors the `_presentation` build logic, counts rows per mod, writes `Modules\ModReady\presentation-row-survey.log`. Gated on `Modules\ModReady\presentation-survey.flag`. Triggered from `OptionsVMMixin.RebuildModList` (originally `OnBeforeInitialModuleScreenSetAsRoot` but moved because that hook only fires once at game-load and the user has to drop the flag before clicking PLAY). Result for steve's modlist: max 28 rows (HomesteadsReloaded), median 14, 9 mods. ROT-class mods (200+) not in current install.

**4c ŌĆö BLOCKED, needs Path ╬▒ work.** The plan was to rewrite the prefab XML in `OptionsPrefabExtensions.cs`: replace the 20-row slot loop with a single `<ListPanel DataSource="@RowList">` + an `ItemTemplate` whose widget variants are gated on `@IsBool` / `@IsNumeric` / `@IsText` / `@IsDropdown` / `@IsButton` / `@IsHeader`. **Blocker** documented at the top of `OptionsPrefabExtensions.cs`:

> v0.4.19: reverted to 10-slot inline pagination (v0.4.16 working pattern). NavigatableListPanel + ItemTemplate failed to iterate CurrentRows twice (v0.4.5 with MCMOptionRow prefab include, v0.4.18 with inline ItemTemplate). UIExtenderEx mixin DataSourceProperty doesn't seem to surface MBBindingList<T> in a way NavigatableListPanel recognizes. Sticking with what works.

So the simple "point the ListPanel at `@RowList`" path was attempted twice in v0.4.5 and v0.4.18 and both times the binding system did not iterate the collection. Hypothesis: our `ModReady.UIExtenderEx` mixin reflection layer surfaces scalar `[DataSourceProperty]` getters into the target VM correctly, but doesn't surface `MBBindingList<T>`-typed getters in a way `ListPanel.ItemTemplate` / `NavigatableListPanel` can iterate.

**4d ŌĆö not started.** Once 4c lands and we've verified the new prefab works end-to-end (header dividers, all 5 widget variants render, scrolling works, mutations stick, preset apply repaints, hint hover still fires), delete the entire `Slot0..Slot19` fan: all `Slot N_X` properties (~25 props ├Ś 20 slots Ōēł 500 LOC), `_slots[]` + `_slotIsHeader[]` + `_slotGroupHeaders[]` arrays, `RefreshSlots()`, `_currentPageIndex` + `_totalPages` + `_selectedPageSummary` + `PaginationVisible` + `Page0_IsCurrent..Page9_IsCurrent`, `ExecutePrevPage` / `ExecuteNextPage`, `ExecuteSlot N_DropdownNext` / `Prev` / `ToggleBool` / `Hover` / `HoverEnd`, the SlotCount=20 const, and the matching slot-row loop in `OptionsPrefabExtensions.cs`. Pure deletion, no behavior change once 4c is verified.

### Why Path ╬▒ (not ╬▓ or ╬│)

The user picked Path ╬▒ (fix the mixin ŌåÆ MBBindingList plumbing in our own UIExtenderEx re-implementation) because we own the entire `ModReady.UIExtenderEx` codebase. The most likely root cause ŌĆö that the mixin's runtime reflection layer doesn't expose `MBBindingList<T>`-typed getters as iterable bindings to the parent ListPanel ŌĆö is fixable in our own source. Timebox: 1 hour, fall back to Path ╬│ (just bump `SlotCount` to 32, hide pagination UI, accept truncation for ROT-class outliers) if root cause isn't found.
