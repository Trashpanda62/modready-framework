# BetaDeps v1.0 — Non-Destructive Music Picker (headline feature)

**Status:** Planning. Both core primitives spike-proven (2026-06-08). Two feature-mechanics spikes (#3, #4) remain before full build.
**Replaces:** the old v1.0 "production-ready compatibility polish" framing in `ROADMAP.md` (Realm of Thrones dropped; compatibility work demoted to ongoing maintenance).
**One-liner:** Let players choose their own music per game context — campaign, battle, siege, settlement, naval — *without* overwriting a single vanilla audio file, the way every existing Nexus music pack forces you to.

---

## 0. Why this is the headline

Every music mod on Nexus today is destructive: you replace `Battle_Medium_02.ogg` on disk and lose the original. No per-context choice, no mixing packs, no toggling, no DLC awareness, and you fight the launcher's file-integrity checks. BetaDeps already owns the hard parts of being non-destructive (Harmony chokepoint patching, additive asset loading, an MCM/Gauntlet UI layer, PatchShield safety). A **bring-your-own (BYO) music picker** is the feature that turns the compatibility framework into something a non-modder *wants installed for its own sake* — and nothing else on Nexus does it.

## 1. What the spikes proved

Two independent runtime primitives, both confirmed in the live game on 2026-06-08.

### Spike #1 — PSAI additive soundtrack + custom theme (PASS)
`runtime.log`, menu context:
```
LoadSoundtrackFromProjectFile(Native, NavalDLC, BetaDeps)  → OK   (3-module additive merge)
TriggerMusicTheme(90001, 1.0)                              → OK   (custom out-of-enum theme accepted + played)
poll t+0.5s: themeId=90001                                       (our theme is live)
poll t+1.5s: themeId=10244 (NavalMainTheme)                      (reverted — see note)
```
- **Proven:** PSAI accepts a custom theme ID outside the vanilla enum, and will play a **loose `.ogg`** referenced from an additive module `soundtrack.xml`. The 3-project merge (Native + NavalDLC + BetaDeps) works.
- **The revert is expected, not a failure.** We called `TriggerMusicTheme` *raw*. The menu's own music handler re-requested its theme on the next state tick and overrode us. **Conclusion: the feature must patch the selector chokepoint, not raw-trigger.** (See §3.)

### Spike #2e — Engine.Music loose-file channel (PASS, in-settlement)
`runtime.log`, inside settlement "Amycon":
```
GetFreeMusicChannelIndex = 0                  (channel 0 FREE in settlement; was -1 at menu)
LoadClip(0, ...\BYO\test.ogg)                 → IsClipLoaded(0) = True
SetVolume(0, 1.0) + PlayMusic(0)              → audible second track confirmed by ear
poll t+0.5s / t+1.5s / t+2.5s: IsClipLoaded(0)=True
```
- **Proven:** `TaleWorlds.Engine.Music.LoadClip(int, string)` plays an arbitrary loose file by absolute path on a free channel — **inside a settlement**, where PSAI relaxes its grip and a channel opens up.
- **Key finding:** at the **main menu** `GetFreeMusicChannelIndex` returns **-1** (PSAI holds every channel). Engine.Music loose-file is therefore **only** viable in low-PSAI contexts (settlements). This is exactly the context we need it for, because settlement music is **FMOD-event-based (music.bank), not PSAI** — so PSAI redirection can't reach it and Engine.Music is the only non-destructive door.

### Net: a deliberate two-path architecture
| Primitive | Where it works | Used for |
|---|---|---|
| PSAI redirect (`StartTheme` patch + additive soundtrack) | every PSAI context | menu, campaign, battle, siege, victory, defeat, naval |
| Engine.Music loose-file channel | settlements only (channel free) | town, village, tavern |

Both hide behind one picker UI and one BYO folder convention. The user never sees the seam.

## 3. PSAI path mechanics (the chokepoint)

**Chokepoint:** `TaleWorlds.MountAndBlade.MBMusicManager.StartTheme(MusicTheme theme, float intensity, bool ...)` — the single method every game system funnels through; internally it calls `PsaiCore.Instance.TriggerMusicTheme((int)theme, intensity)`.

**Harmony prefix on `StartTheme`:**
```
1. Map the incoming MusicTheme enum value → our MusicContext bucket.
2. Is BYO enabled for that context AND does its pool have ≥1 track?
   - NO  → return true (run vanilla StartTheme unchanged).
   - YES → pick next track from the context's PlaybackPool (shuffle/sequential),
           ensure it is a segment under our custom theme id (prefix 9000 + localId),
           call TriggerMusicTheme(customThemeId, intensity) ourselves,
           return false (skip original).
```
Because **we own the chokepoint**, every time the game re-requests a context theme we re-redirect — this is what fixes the Spike #1 revert. PSAI still handles intensity/segment crossfade *within* our custom theme, so adaptive behavior (battle intensity ramps) is preserved for free.

**BYO file delivery into PSAI:** a generated `soundtrack.xml` defines one custom theme per context, each theme holding one segment per user track:
- ModuleIdPrefix = `9000`. Theme localIds 1..N → effective ids `90001..9000N` via the documented `int.Parse(prefix + localId)` string-concat mangling (NOT multiplication).
- Segments reference the user's files by module-relative path (`Music/BYO/<Context>/<file>.ogg`).
- `LoadSoundtrackFromProjectFile(["Native","NavalDLC"(if present),"BetaDeps"])` merges it (Spike #1 proved additive merge).

**Spike #3 PASS (2026-06-08):** 28/28 polls held on themeId=90001, 0 watchdog retriggers needed, 2 distinct segments cycled (90001↔90002) confirming pool cycling. Key findings:
- Real PSAI readiness gate = `Instance != null` **AND** `GetCurrentThemeId() != -1` (not just Instance — the theme must already be playing)
- `MBMusicManager.StartTheme` is NOT called at the menu; menu routes directly through `TriggerMusicTheme`. Patch `TriggerMusicTheme` as the redirect chokepoint (proven sufficient). `StartTheme` may still be the right patch point for in-campaign context switches; leave the StartTheme log-prefix in C2 to confirm.
- Runtime reload open question remains — MVP ships "restart to apply" and upgrades to live reload in polish.

## 4. Settlement path mechanics (Engine.Music)

`SettlementMusicManager`, driven off campaign settlement lifecycle:
```
OnSettlementEntered(MobileParty.MainParty):
   ctx = Town | Village | Tavern (by location)         (note the lords_hall bug, §10)
   pool = BYO pool for ctx + culture (fallback generic)
   if pool empty or BYO disabled → do nothing (vanilla bard plays)
   else:
     suppress vanilla musician FMOD event              (Harmony patch — to confirm Spike #4)
     ch = Music.GetFreeMusicChannelIndex()             (proven free in settlement)
     Music.LoadClip(ch, pool.Next()); Music.SetVolume(ch, trim); Music.PlayMusic(ch)

OnApplicationTick (while in settlement, channel active):
   if current clip finished → Music.LoadClip(ch, pool.Next()); PlayMusic(ch)   (advance pool)

OnSettlementLeft / mission change:
   stop + release channel; un-suppress vanilla musician event
```

**Spike #4 findings (2026-06-08):**

Full confirmed `TaleWorlds.Engine.Music` API (10 static methods):
```
GetFreeMusicChannelIndex()                ← returns 0 in settlement (proven Spike #2e + #4)
LoadClip(int index, string pathToClip)    ← loads .ogg into channel
IsClipLoaded(int index)                   ← check load success
IsMusicPlaying(int index)                 ← True while playing, transitions False at clip end
PlayMusic(int index)                      ← start playback
PauseMusic(int index)                     ← pause (bonus; useful for fade-out)
PlayDelayed(int index, int deltaMs)       ← delayed start (bonus; useful for crossfade timing)
SetVolume(int index, float volume)        ← set channel volume
StopMusic(int index)                      ← stop + seek-to-start
UnloadClip(int index)                     ← explicit unload (use in cleanup)
```

1. **Bard suppression** — `MBMusicManager.StartTheme` is NOT called when entering or while inside a settlement (zero calls logged in 90s settlement visit). Settlement music is triggered via FMOD event directly. Finding the exact call site is a polish task; for MVP ship "plays over vanilla" and suppress in C3 polish.
2. **End-of-clip detection** — no GetPosition/GetDuration. Use `IsMusicPlaying` polling: when it transitions `False`, the clip ended naturally → load+play next track. Proven: 3 timer-forced advances cycled 01→02→01→02 cleanly.
3. **Volume slider** — `BannerlordConfig` float props are gameplay config only; music volume lives in `TaleWorlds.Library` (likely `ManagedOptions` or `NativeOptions`). One reflection lookup to confirm. MVP ships `SetVolume(1.0f)` until found.
4. **Clean release** — `StopMusic(int)` + `UnloadClip(int)` on settlement exit. Confirmed working.

## 5. BYO data model — folder convention

The on-disk folder tree IS the data model. Drop files in, they're picked up. The UI is a view/assignment layer over this, never the source of truth (keeps it fully moddable + sample-pack-friendly).

```
Modules/BetaDeps/Music/BYO/
  Menu/                      *.ogg | *.wav
  Campaign/
    Peace/  War/  Dark/
  Battle/
    Small/  Medium/  Large/      (or flat Battle/ → all sizes)
  Siege/
  Victory/
  Defeat/
  Naval/                     (only meaningful with War Sails; greyed out otherwise)
  Settlement/
    Town/
      _generic/  Empire/  Aserai/  Sturgia/  Vlandia/  Battania/  Khuzait/
    Village/
      _generic/  <culture>/...
    Tavern/
      _generic/  <culture>/...
```
Rules: a context with zero files → vanilla plays (graceful no-op). A culture subfolder with zero files → fall back to `_generic/` → fall back to vanilla. `.ogg` preferred, `.wav` accepted (both proven loadable).

## 6. The picker UI

Headline feature ⇒ it must *feel* like a feature, not a buried settings group. Two staged workstreams:

**UI-A (MVP, fast path) — MCM-backed.** Each context becomes an MCM settings group: `Enable BYO` (bool), `Mode` (Shuffle/Sequential dropdown), `Volume trim` (float slider), plus a read-only `Tracks detected: N` label. Ships the engine fully usable on day one using infra BetaDeps already has (Mod Config screen, sliders, dropdowns).

**UI-B (headline) — dedicated "Music" screen.** Two-pane Gauntlet screen on the same prefab-patch + mixin infra as the MCM tab (`docs/PHASE-3-MCM-UI-PLAN.md` patterns):
- Left pane: context list (with DLC-gated Naval greyed when War Sails absent).
- Right pane: the selected context's detected track pool — track names, a **Preview ▶** button (plays the track through the same primitive it'll use in-game), shuffle/sequential toggle, per-context volume trim.
- Empty state per context: "Drop `.ogg` files in `Music/BYO/<Context>/` and restart" with a copy-path button.

**Recommendation:** build UI-A first (unblocks everything), ship the engine behind it, then build UI-B as the thing that earns the "music picker" headline. Don't gate the v1.0 *release* on UI-B if the engine + UI-A are solid — but UI-B is the marketing centerpiece, so target it.

## 7. Component architecture

```
MusicConfig            reads BYO tree → builds MusicContext→PlaybackPool model;
                       persists per-context settings (enable/mode/volume) via MCM storage;
                       (MVP) loads once at startup; (polish) file-watch for live changes.
        |
        ├── PsaiRedirectManager     generates+loads additive soundtrack.xml from pools;
        |                           installs Harmony prefix on MBMusicManager.StartTheme;
        |                           MusicTheme → MusicContext → customThemeId; track selection.
        |
        └── SettlementMusicManager  hooks settlement enter/leave + tick;
                                    grabs Engine.Music channel; LoadClip/PlayMusic/advance;
                                    suppresses vanilla bard FMOD event.

MusicContext   enum: Menu, CampaignPeace/War/Dark, BattleSmall/Medium/Large, Siege,
               Victory, Defeat, Naval, SettlementTown/Village/Tavern.
PlaybackPool   per-context List<track> + mode + cursor; Next() honors shuffle/sequential.
NavalGate      detects NavalDLC loaded → enables Naval context + UI row.
UI layer       MCM groups (UI-A) and/or dedicated Music screen (UI-B).
```
All built on existing BetaDeps foundation: `DiagLog`, `PatchShield` (auto-wraps our Harmony patches), `SafeBind`, the MCM settings/storage stack, the UIExtenderEx prefab-patch + ViewModel-mixin stack.

## 9. Naval DLC gating

War Sails ships as module **NavalDLC** (PSAI prefix `1024`); `MusicNavalBattleMissionView` in `NavalDLC.View.dll` drives naval combat music through the same PSAI pipeline. So naval BYO is *just another PSAI redirect* — no new mechanic. `NavalGate` checks for the loaded NavalDLC module/assembly at startup:
- Present → Naval context enabled, picker row active, `Music/BYO/Naval/` honored.
- Absent → Naval row greyed ("Requires War Sails DLC"), folder ignored.

Demand is real (Nexus naval-music mods 9431 + 9218 exist), and we're the only non-destructive option.

## 10. Settlement notes

- **`lords_hall` vanilla bug (bonus):** 12 `lords_hall` tracks in `settlement_tracks.xml` never play because the XML keys `location="lords_hall"` while the scene id is `lordshall` (no underscore). Because `SettlementMusicManager` already resolves context by location, we can **make these vanilla tracks audible for the first time** as a no-cost side benefit (map `lords_hall` → `lordshall`). Tangential to BYO; ship as a "we also fixed this" footnote, not core scope.
- **Tavern vs Town:** tavern is a distinct location within a settlement; the manager must distinguish entering the tavern scene from the town menu to switch pools.

## 11. Optional sample pack

Per the grill-me decision: **users supply their own music; we provide an optional royalty-free sample bundle** as a *separate* download (Nexus "miscellaneous files" or a tiny companion mod) that simply drops curated CC0/royalty-free `.ogg` files into the `BYO/` tree. Keeps the core mod small, sidesteps licensing in the main package, and gives newcomers a one-click "hear it working" path. Build after the engine is real (P2).

## 12. Risks & edge cases

- **PSAI revert (CLOSED):** Spike #3 proved `TriggerMusicTheme` holds the redirect for 56s with zero drift at the menu. Patch `TriggerMusicTheme`; also patch `StartTheme` with a log-only prefix in C2 to confirm it fires during in-campaign context switches.
- **Runtime reload:** if Spike #3 shows soundtrack reload needs a restart, ship "restart to apply new tracks" for MVP — acceptable, common in modded Bannerlord.
- **Channel contention:** Engine.Music channel is free *in settlements*; if a future context wants loose-file outside settlements we'd hit the -1 wall — out of scope, PSAI covers those.
- **Format support:** `.ogg` proven; confirm `.wav` and reject/transcode exotic formats gracefully (no CTD on a bad file — wrap in PatchShield/try-catch, log + skip).
- **Mod load order / other music mods:** detect and warn if a *destructive* music mod is also installed (it'll fight us). IncompatibleModDetector already exists — add a soft advisory.
- **Save safety:** music state is runtime-only; nothing serialized into saves → SaveShield surface unaffected.

## 13. Out of scope for v1.0

- Live waveform/scrubbing UI, drag-drop playlist ordering (UI-B polish → v1.1).
- Live file-watch hot-reload (MVP = restart to apply).
- Crossfade authoring / per-track fade curves (PSAI handles intra-theme crossfade; manual curves later).
- Loose-file BYO for non-settlement contexts (PSAI redirect covers them; no need).
- Per-track conditional logic (time of day, weather, faction-specific beyond culture).
- Streaming from URLs / online libraries.
